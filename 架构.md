## 🏗️ RAGFlow 项目架构详解

### **一、整体架构设计**

RAGFlow 采用**微服务架构**，基于 Docker 容器化部署，主要包含以下核心层次：

```
┌─────────────────────────────────────────┐
│          前端层 (Frontend)               │
│     React + TypeScript + UmiJS          │
└──────────────┬──────────────────────────┘
               │ HTTP/WebSocket
┌──────────────▼──────────────────────────┐
│         API 网关层 (API Layer)           │
│      Flask/Quart + Gin (Go)             │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│        业务逻辑层 (Business Layer)       │
│  ┌────────┐ ┌────────┐ ┌────────┐      │
│  │  RAG   │ │ Agent  │ │ DeepDoc│      │
│  │  Core  │ │ Engine │ │ Parser │      │
│  └────────┘ └────────┘ └────────┘      │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│        基础设施层 (Infrastructure)       │
│ MySQL | ES/Infinity | Redis | MinIO    │
└─────────────────────────────────────────┘
```

### **六、核心工作流程**

1. **文档上传** → DeepDoc 解析 → 文本提取 → 向量化 → 存入 ES/Infinity
2. **用户查询** → Query 理解 → 向量检索 + 全文检索 → Rerank 排序 → LLM 生成 → 返回答案
3. **Agent 工作流** → 画布编排 → 组件执行 → 工具调用 → 结果输出

---

### **七、架构特点**

✅ **模块化设计**: 各组件职责清晰，易于扩展  
✅ **多语言混合**: Python 处理 AI 任务，Go 处理高并发  
✅ **容器化部署**: 一键部署，环境隔离  
✅ **可扩展性**: 支持多种 LLM、Embedding、数据库后端  
✅ **深度文档理解**: 自研 DeepDoc 引擎处理复杂文档  
✅ **可视化编排**: Agent 工作流图形化配置  

这个架构设计使得 RAGFlow 能够灵活处理各种非结构化数据，并提供强大的检索增强生成能力。
